Subnet separation: ECS tasks (10.0.3–4.0/24) and RDS (10.0.5–6.0/24) are now in dedicated subnets. ECS subnets route outbound via NAT Gateways (needed for ECR pulls, CloudWatch). DB subnets have no outbound route — only the implicit VPC-local route applies. RDS has no reason to reach the internet.

Two isolation layers: (1) Separate subnets with different route tables — a compromised ECS task cannot use the DB subnet's routing. (2) RDS Security Group only allows :5432 ingress from ECSSecurityGroup — even within the VPC, nothing else can reach RDS.

DB credentials: DB_PASSWORD is never stored in the task definition. It is injected at task startup by the ECS agent pulling the value from Secrets Manager using the Secrets field. The DATABASE_URL is assembled in config.ts at process start.

Cost impact: Two NAT Gateways ~$64/mo · RDS db.t3.micro ~$13/mo · 4 extra subnets/route tables — negligible. For dev/staging you can use a single NAT Gateway to halve the NAT cost.
